# Tarkov Hideout Tracker

Небольшое веб-приложение, чтобы следить, какие предметы ещё нужно собрать для
прокачки модулей убежища в Escape from Tarkov. 
## Стек

- React + Vite + обычный JavaScript (без TypeScript)
- Material-UI с кастомной тёмной темой в стиле Tarkov
- Состояние в `localStorage` (ключ `tarkov-hideout-v1`)
- Данные берутся из статического `src/data/data.json`, собранного из Excel
- Деплой как статика на Render (см. `render.yaml`)

## Запуск

```bash
npm install
npm run dev        # дев-сервер на http://localhost:5173
npm run build      # сборка в dist/
npm run preview    # посмотреть собранную статику
```

## Данные

`src/data/data.json` собирается из приложенного Excel-файла
`tarkov_hideout_requirements_2026-06-14.xlsx` (лист «Требования»):

```bash
npm run build:data        # то же самое, что python scripts/build_data.py
```

Парсер (`scripts/xlsx_reader.py`) читает xlsx стандартной библиотекой Python,
без openpyxl. Готовый json коммитится в репозиторий, в рантайме Excel уже не
открывается.

### Проверки корректности

```bash
python scripts/verify_summary.py   # свод приложения сходится с листом «Свод предметов»
node   scripts/smoke_test.mjs      # логика агрегации и готовности уровней
node   scripts/ssr_smoke.mjs       # весь App рендерится без ошибок
```

`verify_summary.py` сравнивает свод, который считает приложение (при всех
`builtLevels = 0` и пустом `collected`), с эталонным листом «Свод предметов»:
суммы, разбивка на FIR и обычные, список «где используется» совпадают. В эталоне
учтены только обязательные предметы, поэтому опциональные (4 строки Fleece fabric
в Defective Wall) там не считаются. В приложении они показываются с бейджем «опц.».

## Модель состояния (самое важное)

Данные делятся на две части:

- **Статика** (`data.json`): модули, уровни, предметы, условия. Не меняется.
- **Состояние пользователя** (localStorage), всего два объекта:
  - `builtLevels = { moduleId: builtLevel }`, какой уровень модуля уже построен
    (0 значит не построен);
  - `collected = { itemKey: true }`, где `itemKey = name + (fir ? "__FIR" : "__STD")`.

Статус «найдено» привязан к самому предмету (название плюс FIR), а не к строке
требования, и он общий для всех модулей. Поэтому отметка в окне «Свод» сразу
видна во всех модулях окна «Модули» и наоборот, отдельную синхронизацию писать
не пришлось, это буквально одно и то же значение. FIR-версия и обычная версия
одного предмета это разные ключи (разные слоты): FIR нельзя зачесть вместо
обычного.

Всё остальное считается на лету, не хранится: список к сбору это предметы
уровней строго выше построенного (`level > builtLevels[id]`), а уровень готов к
постройке, когда отмечены все его обязательные предметы (опциональные не блокируют).

## Деплой на Render

Static Site:

- Build command: `npm install && npm run build`
- Publish directory: `dist`
